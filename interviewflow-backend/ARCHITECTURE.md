# InterViewFlow AI - Architecture Documentation

## System Philosophy

**This is not a chatbot. This is an AI Evaluation Engine.**

InterViewFlow AI is an **Adaptive, Composable AI Interview Engine** that simulates real technical and behavioral interviews and produces hiring-grade evaluation reports.

The system treats an interview as a **closed-loop reasoning pipeline**, not a static Q&A flow.

## Core Architecture

### Interview Pipeline

```
Role → Context → Question Generation → Memory → Evaluation → Analytics → Adaptation → Final Report
```

Each block operates independently but shares state through a central **Interview State** object.

## 1. Interview State (Central Brain)

**Location:** `models/Interview.js`

The Interview State is the living cognitive state that drives all adaptive decisions. It maintains:

- **Role & Target Position**: Frontend Intern, Backend Engineer, System Designer, etc.
- **Skill Focus Areas**: Technical and behavioral skills being tested
- **Experience Level**: Intern, Junior, Mid-Level, Senior, Staff+
- **Current Difficulty Level**: Easier, Same, Harder
- **Full History**: All questions and answers
- **Per-Turn Evaluation Metrics**: Multi-dimensional scores
- **Detected Strengths & Weaknesses**: Pattern recognition
- **Confidence & Clarity Trends**: Performance over time
- **Concept Mastery Map**: Mastery level per concept (0-10)
- **Performance Progression**: Trajectory, improvement rate, consistency

### Key Fields

```javascript
{
  cognitiveState: {
    currentDifficulty: 'same' | 'easier' | 'harder',
    performanceTrend: 'improving' | 'stable' | 'declining' | 'volatile',
    conceptMastery: Map<concept, masteryLevel>,
    confidenceTrend: [scores],
    clarityTrend: [scores],
    detectedStrengths: [{ concept, evidence, confidence }],
    detectedWeaknesses: [{ concept, evidence, severity }],
    communicationProfile: { clarity, structure, completeness, consistency },
    reasoningQuality: { average, trend },
    learningCurve: { slope, variance, cognitiveLoad, trend }
  },
  dimensionScores: {
    relevance, correctness, depth, clarity, confidence, reasoning
  },
  skillScores: Map<skill, { averageScore, turnCount, trend }>,
  adaptationHistory: [{ turn, decision, reason, previousScore, newDifficulty }]
}
```

## 2. Question Generation (Cognitive Probing)

**Location:** `services/blocks/question.block.js`

Questions are **never random or static**. Every question is generated by reasoning over:

- Role expectations
- Skill being evaluated
- Current difficulty level
- Weak concept areas
- Strong concept areas
- Past questions already asked
- Quality and depth of previous answers
- Whether candidate is improving or struggling

### Adaptation-Driven Generation

The question generator receives:
1. **Context**: Role, skill, level, previous questions, weak/strong areas
2. **Adaptation Decision**: From the adaptation block (what strategy to use)
3. **Cognitive State**: Current performance trends and mastery levels

### Question Types

- **Foundational**: When understanding is shallow
- **Deep**: "Why", "how", and "trade-off" questions when strong
- **Follow-up**: Targeted questions on same concept after mistakes
- **Scenario-based**: When answers are vague
- **Edge Cases**: When performance is strong

Each question includes:
- Clear intent (what concept it's testing)
- Defined difficulty
- Expected concept set
- Reasoning (why it was chosen at that moment)

## 3. Answer Validation & Evaluation

**Location:** `services/blocks/evaluation.block.js`

Each answer is analyzed along **multiple dimensions**:

### Dimensions

1. **Relevance** (0-10): Does the answer address the question?
2. **Factual Correctness** (0-10): Are technical statements accurate?
3. **Conceptual Depth** (0-10): Does user understand internals or only surface?
4. **Clarity & Structure** (0-10): Is explanation coherent and logical?
5. **Confidence & Precision** (0-10): Is user decisive or vague?
6. **Completeness** (0-10): Are key concepts missing?
7. **Reasoning Quality** (0-10): Does user explain "why", not just "what"?

### Detection Capabilities

- Partial correctness
- Misconceptions
- Hallucinations
- Memorized but shallow answers
- Conceptual gaps
- Overconfidence without understanding

### Output

Each evaluation produces:
- Multi-dimensional scores
- Qualitative assessment
- List of strengths
- List of missing concepts
- List of mistakes
- Targeted improvement feedback

## 4. Memory (Long-Term Reasoning)

**Location:** `services/memory.js`, `models/Memory.js`

Memory is **structured cognitive state**, not just storage.

### Per-Turn Storage

- Question intent and expected concepts
- User answer
- Evaluation metrics (all dimensions)
- Mistakes and gaps
- Strength signals
- Confidence level
- Topic and concept tags
- Adaptation decision that led to this question

### Cross-Turn Analysis

Memory enables derivation of:
- Knowledge trends (improving, stagnating, declining)
- Repeated weak areas
- Repeated strong areas
- Concept mastery levels
- Communication consistency
- Confidence growth or drop

## 5. Adaptive Control Logic

**Location:** `services/blocks/adaptation.block.js`

After every evaluation, the system decides:

- **Difficulty**: Easier, same level, or harder?
- **Topic**: Go deeper into same topic or switch?
- **Focus**: Fundamentals or edge cases?
- **Type**: Theory or real-world scenarios?
- **Probe**: Explanation or problem-solving?

### Decision Factors

- Recent performance trajectory
- Variance in scores
- Gap severity
- Role expectations
- Interview progression stage

### Adaptation Decisions

- `increase_difficulty`: Candidate is strong
- `decrease_difficulty`: Candidate is struggling
- `maintain_difficulty`: Performance is stable
- `switch_topic`: Need to test different areas
- `deepen_probe`: Go deeper into same concept
- `clarify_fundamentals`: Focus on basics
- `challenge_edge_case`: Test advanced understanding
- `scenario_based`: Use real-world scenarios

## 6. Analytics & Scoring System

**Location:** `services/blocks/analytics.block.js`

Scores are **not arbitrary**. The system computes:

### Per-Dimension Scores

- Relevance, Correctness, Depth, Clarity, Confidence, Reasoning
- All normalized to 0-10 scale

### Per-Skill Aggregates

- Average score per skill
- Turn count per skill
- Trend (improving/stable/declining)

### Consistency Metrics

- Score variance
- Stability over time
- Consistency score (0-10)

### Learning Curve Analysis

- Slope (improvement rate)
- Variance (consistency)
- Cognitive load handling
- Trend (improving/stable/declining/volatile)

### Normalization

All scores are normalized against role level expectations:
- Intern: Lower expectations
- Junior: Moderate expectations
- Mid-Level: Standard expectations
- Senior: High expectations
- Staff+: Expert expectations

## 7. Coaching Logic

**Location:** `services/blocks/coaching.block.js`

Generates **actionable, personalized coaching** based on:

- Observed patterns (not single answers)
- Concept mastery gaps
- Communication weaknesses
- Reasoning quality issues
- Specific mistakes and misconceptions

### Coaching Output

- **Critical Gaps**: Most important gaps with evidence
- **Improvement Roadmap**: Prioritized action items with:
  - Specific steps
  - Practice strategies
  - Resources
  - Timeline
  - Success metrics
- **Strengths to Leverage**: How to use what they're good at
- **Communication Coaching**: Specific communication improvements
- **Reasoning Coaching**: How to improve reasoning quality
- **Study Plan**: Week-by-week improvement plan

## 8. Final Hiring-Style Report

**Location:** `services/blocks/report.block.js`

Produces a report as if written by a hiring committee:

### Report Sections

1. **Executive Summary**
   - Overall readiness score (0-100)
   - Hire recommendation (Strong Hire / Hire / Borderline / No Hire)
   - One-paragraph summary
   - Confidence level

2. **Performance Breakdown**
   - Technical depth assessment
   - Problem-solving maturity
   - Communication effectiveness
   - Reasoning quality
   - Consistency

3. **Skill Assessment**
   - Per-skill performance with evidence
   - Strongest areas with examples
   - Weakest areas with examples
   - Skill gaps and impact

4. **Strengths & Weaknesses Clusters**
   - Grouped by theme
   - Evidence-based
   - Role alignment analysis

5. **Communication Assessment**
   - Clarity, structure, completeness, consistency
   - Improvement areas

6. **Problem-Solving Maturity**
   - Approach, depth, trade-offs, edge cases

7. **Learning Potential**
   - Trajectory, adaptability, growth mindset

8. **Risk Factors**
   - Critical gaps, consistency concerns, red flags
   - Mitigation strategies

9. **Hire Recommendation**
   - Verdict with detailed justification
   - Confidence level
   - Comparison to role expectations

10. **Improvement Roadmap**
    - Prioritized actions
    - Timeline
    - Success metrics

## 9. Composability

Each part of the system is **modular**:

- **Question Generator**: Can be replaced
- **Evaluation Logic**: Can be upgraded
- **Memory Strategy**: Can be extended
- **Scoring Rubric**: Can be tuned
- **Report Generator**: Can be customized per company

### Future Extensions

The architecture supports reuse for:
- Coding interviews
- HR interviews
- Leadership assessments
- Skill certification
- AI coaching platforms

## 10. Interview Service (Orchestration)

**Location:** `services/interview.js`

The Interview Service orchestrates the entire pipeline:

### Flow

1. **Start Interview**
   - Initialize cognitive state
   - Generate first question
   - Create interview document

2. **Process Answer** (The Core Adaptive Loop)
   - Evaluate answer (multi-dimensional)
   - Save to memory
   - Update cognitive state
   - Compute analytics
   - Decide adaptation strategy
   - Generate next question based on adaptation
   - Update interview state

3. **End Interview**
   - Compute final analytics
   - Generate coaching plan
   - Generate final report
   - Save everything

## Data Flow

```
User Answer
    ↓
Evaluation Block (Multi-dimensional analysis)
    ↓
Memory Service (Save turn with full context)
    ↓
Update Cognitive State (Performance trends, strengths/weaknesses)
    ↓
Analytics Block (Compute all metrics)
    ↓
Adaptation Block (Decide next strategy)
    ↓
Question Block (Generate next question with adaptation)
    ↓
Update Interview State
    ↓
Return to User
```

## Key Design Principles

1. **State-Driven**: Everything flows through the central Interview State
2. **Adaptive**: Every decision is based on observed patterns
3. **Explainable**: Every question, evaluation, and decision has reasoning
4. **Composable**: Each block can be replaced or extended
5. **Production-Ready**: Error handling, fallbacks, validation
6. **Scalable**: Modular architecture supports growth

## System Philosophy Recap

- **Questions are probes**: Designed to test specific concepts
- **Answers are signals**: Multi-dimensional analysis reveals truth
- **Memory is intelligence**: Patterns emerge from structured history
- **Scores are diagnostics**: Not arbitrary, but meaningful metrics
- **Adaptation is control**: Real-time strategy adjustment
- **Reports are decisions**: Hiring-grade evaluation documents

This system simulates how a real interviewer thinks, evaluates, adapts, and concludes — in a structured, explainable, and production-ready way.
